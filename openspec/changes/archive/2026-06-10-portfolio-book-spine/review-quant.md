# Quant-Math Code Review — portfolio-book-spine

**Scope:** the quant/accounting core of the `portfolio-book-spine` refactor (diff vs
`main`): the netted book (`core/portfolio_foundation.py`), the evaluation per-fold
projection (`evaluation/_spine_metrics.py`), and the validation ledger
(`validation/engine_backend.py`). Read-only. Methodology: `quant-math-code-review`
SKILL. The two deferred test failures and the `foundation_enabled`/VBT-build
residuals were checked only for hidden quant leaks (none found there — see note F-NOTE).

## Verdict

**Correct with fixes.** The core money model is sound: per-symbol netting, the
cash + unrealized-PnL equity decomposition, cost-on-delta, the funding sign/window/
dedup, the NAV↔ledger reconciliation *for flat-ending books*, and the statistical
machinery (Mertens/Lo Sharpe SE, Bailey–López de Prado DSR, AR(1) effective-N) are
all implemented correctly and verified numerically. The blockers/majors are at the
**boundaries between the NAV truth and the two derived numbers that are actually
gated** — the realized-only `net_return` diverges from NAV on non-flat-ending
windows, and the evaluation/at-risk discipline is not applied consistently across
surfaces.

## What is quant-correct (preserve)

Verified by hand-trace and by running the real book:

- **Per-symbol netting + cost-on-delta** (`_apply_decision`): trades only `delta = target_qty − current`, charges `|delta·fill|·c` once. Flat/close/reverse/trim branches are correct; the reversal attributes the single crossing trade's cost to the closed leg and re-opens with `open_cost=0` (cost counted once in cash; the close/open split is attribution only). Idempotent re-emission is a true no-op. **Verified** reconciliation for long-with-costs and for a long→short→flat reversal.
- **Equity decomposition** (`_equity_at_mark`): `cash + Σ(signed_qty·mark − cost_basis)` = cash + unrealized PnL. Correct because `cash` was already debited the cost basis at open. Same-bar entries are sized against one pre-entry snapshot, so intended gross = Σ|weight| independent of fill order (spec requirement met; `test_offsetting_legs...` passes).
- **NAV↔ledger reconciliation (flat-ending)** (`_close_leg`): `final_nav − INITIAL == Σ realized_pnl` holds exactly for flat-ending books, including multi-symbol + funding and reversals. (See finding #1 for the non-flat case.)
- **Funding** (`_apply_funding`): sign `−signed_qty·mark·rate` (long pays, short receives) — correct; window `entry < funding_ts ≤ now` is strict on the lower bound, so no funding is charged for the period before the position existed (a funding event whose timestamp equals the entry-fill bar is correctly **not** charged — verified). Dedup by `(symbol, funding_ts)` with `funding_rates_match` and conflicting-rate detection is correct; `funding_cashflow` is reset on `_open_leg` so a reversal does not carry prior funding.
- **RiskRule** (`_risk_rule_fired`, `_update_trailing`): stop/take-profit/trailing directionality (entry vs peak/trough) is correct; end-of-bar printed-mark fill with intrabar OHLC realism explicitly deferred (matches the declared contract, D2/F5). The re-entry latch (`latched`) correctly suppresses standing-target re-entry until a different target arrives, and the latch is honored in both `_apply_decision` and `_check_intended_budget`.
- **Fail-closed budget** (`_check_intended_budget`): intended gross/net measured on declared standing target weights (not live drift — D3); per-symbol dict overwrite prevents double-counting a held+re-decided symbol; precedence leverage-budget → unfinanced is correct; `unfinanced_leverage` correctly exempts modeled-financing kinds (`crypto_perp_funding`) and fires for net>1 otherwise. Live marked gross/net is a reported series, never a breach (verified `test_equity_fully_invested...`). `_scenario_feasibility` precedence (leverage > zero-cost > insufficient-samples) is correct.
- **Statistics:** Sharpe SE numerator `1 − skew·SR + ((kurt−1)/4)·SR²` **= Mertens/Lo** form with non-excess kurtosis (algebraically `1 + 0.5SR² + (kurt−3)/4·SR²`); the `_shape` kurtosis is non-excess (E[z⁴], normal=3) and the SE default `kurtosis_value=3.0` is consistent. DSR expected-max-Z `(1−γ)Φ⁻¹(1−1/N) + γΦ⁻¹(1−1/(Ne))` matches Bailey–López de Prado (N=10 → 1.575, as expected); `dsr = Φ((SR−threshold)/SE)` with `threshold = benchmark + SE·E[maxZ]`. Per-bar (non-annualized) SR with per-bar SE is internally consistent (annualization would cancel in the ratio). `trial_count=1 → threshold=benchmark` (no inflation). Min-sample gate and `effective_sample_size` clamp `[1, n]` prevent degenerate denominators.
- **Subwindow bucketing:** half-open `[left, right)` with last bucket inclusive; a return spanning a seam is attributed once to the later window — no double-count, no drop. Full-train stats built from the concatenation of per-bucket at-risk chunks equal the whole-path at-risk sample.
- **Causality:** decisions fill at `decision_index + entry_lag_bars` (default 1 → next bar, conservative; lag 0 = decide-and-fill-at-observed-close, a standard zero-latency convention, not look-ahead since `as_of_time ≤ decision_time`). No same-bar future-price fill.
- **Runner gating:** `succeeded = completed and failure_stage is None`; a raised `FeasibilityError` or an infeasible verdict maps to a `feasibility` failure_stage → `succeeded=False`. Verified fail-**closed** (not fail-open) including the `foundation_enabled=False` path (returns a feasibility failure, not a swallowed success).
